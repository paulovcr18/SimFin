<objective>
Remove the broken scenario management UI from the PWA and fix silent error swallowing in `authOnLogin`.

Purpose: The "Salvar Cenário" button and dropdown in the topbar promise multi-scenario management that does not exist. `scenarioSave()` only stores `{ name, createdAt, updatedAt }` — not the actual simulation inputs — so the feature is an illusion. Showing UI that promises a broken feature erodes trust.

Separately, all app-init calls in `authOnLogin` are wrapped in bare `catch(e) {}` blocks with no error logging, making silent failures invisible during debugging.

Output:
- `index.html`: scenario management UI block (`#scenarioWrap` div, lines ~108–133) removed from the topbar
- `js/auth.js`: all `catch(e) {}` in `authOnLogin` updated to `catch(e) { console.error('[init]', e); }`
- `js/auth.js`: `scenarioAutoTouch()` call removed from `authOnLogin` (no scenario UI to update)
- `js/storage.js`: scenario functions `scenarioSave`, `scenarioBtnUpdate`, `scenarioAutoTouch`, `toggleScenarioDrop`, `closeScenarioDrop` are KEPT (they are referenced by `js/db.js` and `localStorage` migration), but any UI-only helper that solely updates the now-deleted DOM elements can be left as no-ops. Do NOT delete the functions — just note they are now orphaned UI helpers.
</objective>
